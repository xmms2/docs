193 /*
194  * drawtime: expected time needed to process the data in milliseconds after collecting it
195  if >= 0, the data is returned as soon as currenttime >= (playtime - drawtime);
196  data is thrown away if playtime < currenttime, but not if playtime < currenttime - drawtime
197  if < 0, the data is returned as soon as available, and no old data is thrown away
198  * blocking: time limit given in ms to wait for data. The process will sleep until new data is available, or the
199  limit is reached. But if data is found, it could still wait until it is current (see drawtime).
200  * returns size read on success, -1 on failure (server killed!) and 0 if no data is available yet (retry later)
201  * if a signal is received while waiting for data, 0 is returned, even before time ran out
202  * Note: the size read can be less than expected (for example, on song end). Check it!
203  */
