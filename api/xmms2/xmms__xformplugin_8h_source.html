28 /**
29  * @defgroup XForm XForm
30  * @ingroup XMMSServer
31  * @brief XForm API
32  *
33  * An xform (transform) is something that reads data and applies some
34  * kind of transformation to it such as decoding or demuxing or
35  * applying an effect.
36  *
37  * The xform api is designed to allow xforms to be connected in a
38  * chain where each xform does a different transformation step. Each
39  * xform provides a "read" method, which should return transformed
40  * data and when it needs more input data, it should call the read
41  * method of the previous xform in the chain.
42  *
43  * The type of the data flowing from one xform to another is described
44  * by an xmms_stream_type_t. So an xform registers which
45  * xmms_stream_type_t it wants as input and when initialised it tells
46  * what type the output data is. This allows the chain of xforms to
47  * easily be built.
48  *
49  * @{
50  */
